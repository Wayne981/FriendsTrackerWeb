Code Explanation

function setupAuthListener() {
  supabaseClient.auth.onAuthStateChange((event, session) => {
    console.log("Auth State changed:", event);
    switch(event) {
      case 'SIGNED_IN':
        if(session) {
          currentUser = session.user;
          resetLoadingStates();
          showMainApp();
        }
        break;

      case 'SIGNED_OUT':
        currentUser = null;
        showAuthModal();
        clearFriendsList();
        break;

      case 'TOKEN_REFRESHED':
        if(session) {
          currentUser = session.user;
        }
        break;
    }
  });
}

What this code does:

  • Sets up an authentication state listener that triggers when the user's auth state changes
  • Handles three different events: SIGNED_IN, SIGNED_OUT, and TOKEN_REFRESHED
  • When a user signs in, it stores the user data, resets UI states, and shows the main app
  • When a user signs out, it clears user data, shows the auth modal, and clears the friends list
  • When the token is refreshed, it updates the current user data

Common Events

  • SIGNED_IN: User successfully authenticates
  • SIGNED_OUT: User signs out or session expires
  • TOKEN_REFRESHED: Authentication token is renewed
  • USER_UPDATED: User details are changed
